# 3.2 重构 Anthropic

## 背景与目标

第一章你在 `src/anthropic.ts` 里实现了一个独立函数 `callAnthropic()`。现在把它封装成一个实现 `LLMProvider` 接口的类。

这次重构是**纯搬运**——逻辑几乎不变，只是从函数变成类方法，同时把 API key 的管理收到构造函数里。

**本节结束后，你应该有：**
- `src/providers/anthropic.ts`：`AnthropicProvider` 类，实现 `LLMProvider` 接口
- 原来的 `src/anthropic.ts` 可以删除（或保留做参考）

---

## Claude Code 是怎么做的

`references/claude-code/src/services/api/claude.ts` 是 Claude Code 的 Anthropic 调用层。它处理了多种 Provider（Bedrock/Vertex/Foundry）、重试、token 统计等复杂逻辑，但核心的流式事件处理和我们第一章实现的是同一个逻辑。

我们只需要把 `callAnthropic()` 的函数体搬进类里，加上 `implements LLMProvider` 声明。

## 动手实现

### 创建 `src/providers/anthropic.ts`

```ts
import Anthropic from '@anthropic-ai/sdk'
import type { LLMProvider, LLMStreamEvent } from '../core/provider.ts'
import type { Message, Tool } from '../types/message.ts'
// ... 其他 import

export class AnthropicProvider implements LLMProvider {
  private client: Anthropic

  constructor() {
    this.client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })
  }

  async *call(
    messages: Message[],
    tools: Tool[],
    systemPrompt: string,
    model: string,
  ): AsyncGenerator<LLMStreamEvent> {
    // 把原来 callAnthropic() 的函数体搬到这里
    // this.client 替代原来的 client 局部变量
  }
}
```

**实现细节（和第一章 `callAnthropic()` 完全一致）：**

1. **把 `Message[]` 转成 Anthropic SDK 格式**：
   - `TextBlock` → `{ type: 'text', text }`
   - `ToolUseBlock` → `{ type: 'tool_use', id, name, input }`
   - `ToolResultBlock` → `{ type: 'tool_result', tool_use_id, content, is_error }`
   - `user` 消息里的 `ToolResultBlock` 数组需要被包在 `{ role: 'user', content: [...] }` 里

2. **把 `Tool[]` 转成 Anthropic tools 格式**：
   - `{ name, description, input_schema: z.toJSONSchema(tool.inputSchema) }`

3. **调用 `this.client.messages.stream()`，处理流式事件，yield `LLMStreamEvent`**

### 更新 `bin/index.ts`

加一个 `createProvider()` 工厂函数：

```ts
function createProvider(): LLMProvider {
  const providerName = process.env.STEVEN_PROVIDER ?? 'anthropic'
  
  switch (providerName) {
    case 'anthropic':
      return new AnthropicProvider()
    default:
      throw new Error(`Unknown provider: ${providerName}. Supported: anthropic`)
  }
}
```

现在还只支持 `anthropic`，下一节加入 `openai`。
