# 1.5 Engine + CLI 串联

## 背景与目标

`query()` 是纯粹的循环逻辑，它不管理会话状态、不知道用户从哪里来。这节加两层胶水：

- **`Engine`**：管理多轮对话的消息历史，提供 `runOnce()`（单次）和 `chat()`（多轮）接口
- **`bin/index.ts`**：CLI 入口，解析参数，选择运行模式

**本节结束后，第一章目标达成：**
```bash
ANTHROPIC_API_KEY=sk-ant-... bun run bin/index.ts -p "用中文介绍一下你自己"
```
能看到真实的 Claude 流式回复。

---

## Engine 的职责

`query()` 每次调用都是无状态的——你传入消息列表，它执行 loop，结束。

**但 REPL 模式需要跨轮次记忆**：第二轮对话需要包含第一轮的历史。`Engine` 就是管理这个历史的地方。

对标 Claude Code 的 `QueryEngine.ts`——打开 `references/claude-code/src/QueryEngine.ts`，它管理了会话历史、compaction（长对话压缩）、file history snapshot 等。我们只需要实现核心的历史管理。

## 动手实现

### `src/core/engine.ts`

```ts
class Engine {
  private history: Message[] = []

  constructor(private opts: {
    tools: Tool[]
    systemPrompt: string
    model?: string
  }) {}

  // one-shot：执行一次，返回最终文本
  async runOnce(userInput: string): Promise<string>

  // multi-turn：追加到历史，执行，流式回调
  async chat(
    userInput: string,
    onEvent?: (event: QueryEvent) => void
  ): Promise<string>

  clearHistory(): void
}
```

**`runOnce()` 实现逻辑：**
1. 构造 `[{ role: 'user', content: userInput }]`
2. 调用 `query()`，收集所有 `text` 事件拼成字符串
3. 返回最终文本

**`chat()` 实现逻辑：**
1. 把 `userInput` 追加到 `this.history`
2. 用当前 `this.history` 调用 `query()`
3. 把每个事件传给 `onEvent` 回调
4. 把本轮的 assistant 回复追加到 `this.history`
5. 返回最终文本

> **注意**：`chat()` 需要把 agent loop 期间产生的所有消息（包括 tool_result）也追加到历史。否则下一轮对话时 LLM 看不到工具调用记录，上下文会断裂。
>
> 怎么做到这一点？`query()` 内部维护的 `messages` 数组在 loop 结束后已经包含了完整历史，可以通过某种方式把它传出来。思考一下 `query()` 的返回值或者参数设计。

### `bin/index.ts`

CLI 入口，功能：

1. **解析参数**
   - `-p` / `--print`：one-shot 模式
   - `--model`：指定模型（默认 `claude-opus-4-5`）
   - 剩余参数拼成 inline prompt

2. **检测运行模式**
   ```
   有 inline prompt 或 -p 标志 → one-shot 模式
   stdin 不是 TTY（pipe）      → one-shot 模式，从 stdin 读 prompt
   无参数、stdin 是 TTY        → REPL 模式
   ```

3. **one-shot 模式**
   - 读取 prompt（从参数或 stdin）
   - 调用 `engine.runOnce(prompt)`
   - 流式输出到 stdout（使用 `process.stdout.write()` 而不是 `console.log`，避免多余换行）
   - 结束后退出

4. **REPL 模式**
   - 用 `readline.createInterface` 创建交互式输入
   - 循环：读取用户输入 → `engine.chat()` → 打印回复
   - 工具执行时打印 `[Running ToolName...]`
   - Ctrl+C 退出

5. **系统 prompt**

   第一章先用一个简单的 system prompt：
   ```
   You are a coding assistant. Current working directory: {process.cwd()}
   ```

6. **错误处理**
   - 读取 `ANTHROPIC_API_KEY`，未设置则打印错误退出

## 验证

**One-shot 模式：**
```bash
ANTHROPIC_API_KEY=sk-ant-... bun run bin/index.ts -p "用一句话介绍 TypeScript"
```
能看到流式文字输出。

**REPL 模式：**
```bash
ANTHROPIC_API_KEY=sk-ant-... bun run bin/index.ts
```
能进入交互模式，多轮对话，第二轮能记住第一轮的内容。

**Pipe 模式：**
```bash
echo "2+2等于几" | ANTHROPIC_API_KEY=sk-ant-... bun run bin/index.ts -p
```
