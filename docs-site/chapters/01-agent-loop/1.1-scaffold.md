# 1.1 项目脚手架

## 背景与目标

在写任何业务代码之前，先搭好地基：项目结构、TypeScript 配置、依赖管理。

**本节结束后，你应该有：**
- 一个可以运行 `bun run bin/index.ts` 的空项目骨架
- 正确的目录结构，和后续章节完全对应

---

## 为什么用 Bun？

Node.js 运行 TypeScript 需要额外的编译工具（ts-node / tsx / esbuild）。Bun 原生支持 TypeScript，直接执行 `.ts` 文件，零配置，无编译步骤。

这也是 Claude Code 的选择——查看 `references/claude-code/CLAUDE.md`，第一行就写道：

> **Runtime**: Bun (not Node.js). All imports, builds, and execution use Bun APIs.

---

## 目录结构

按照设计文档，`steven-cli-typescript/` 的完整结构如下：

```
steven-cli-typescript/
├── bin/
│   └── index.ts          # CLI 入口
├── src/
│   ├── types/
│   │   ├── message.ts    # 消息类型体系
│   │   └── tool.ts       # 工具接口类型
│   ├── core/
│   │   ├── query.ts      # Agent loop
│   │   └── engine.ts     # 会话管理
│   └── anthropic.ts      # Anthropic 直连（第一章临时实现）
├── package.json
└── tsconfig.json
```

> **注意：** 第一章我们不做 Provider 抽象。`anthropic.ts` 直接放在 `src/` 下，第三章重构时再迁移到 `src/services/api/`。

## 动手实现

### 1. 创建 `package.json`

需要包含：
- `"type": "module"` — Bun 推荐 ESM，后续 `import` 语句需要
- `scripts`：至少有 `"dev": "bun run bin/index.ts"` 和 `"test": "bun test"`
- `dependencies`：`@anthropic-ai/sdk`、`zod`
- `devDependencies`：`@types/bun`

**参考：** `references/claude-code/package.json` 查看真实项目的依赖结构。

### 2. 创建 `tsconfig.json`

关键配置项：
- `"target": "ESNext"`
- `"module": "ESNext"`
- `"moduleResolution": "bundler"` — Bun 专用的模块解析策略
- `"strict": true`

**参考：** `references/claude-code/tsconfig.json`

### 3. 安装依赖

```bash
cd steven-cli-typescript
bun install
```

### 4. 创建目录结构

按上面的树状图，创建所有目录。文件暂时留空，后续章节逐一填充。

### 5. 创建一个最简单的 `bin/index.ts` 验证环境

```ts
console.log('steven-cli starting...')
```
