# 1.3 Anthropic 直连

## 背景与目标

在设计任何抽象之前，先**直接接入 Anthropic SDK**，让 Agent 能真正和 Claude 对话。

这节刻意不做 Provider 接口抽象——我们要先感受"直连"的样子，才能在第三章理解"为什么需要抽象"以及"抽象的代价"。

**本节结束后，你应该有：**
- `src/anthropic.ts`：一个能流式调用 Claude API 并处理工具调用事件的函数
- 能在本地跑通一次真实的 Claude 对话（即使只是 "say hello"）

---

## Anthropic API 的流式响应结构

Anthropic 的 streaming API 是**事件流**，而不是一次性返回完整响应。你需要处理的核心事件：

```
content_block_start   → 开始一个内容块（文本或工具调用）
content_block_delta   → 内容块的增量数据（文本片段 或 工具参数片段）
content_block_stop    → 内容块结束
message_stop          → 整个消息结束
```

**文本内容的流式过程：**
```
content_block_start  (type: 'text')
content_block_delta  (delta.type: 'text_delta', delta.text: 'H')
content_block_delta  (delta.type: 'text_delta', delta.text: 'ello')
content_block_stop
```

**工具调用的流式过程：**
```
content_block_start  (type: 'tool_use', id: 'toolu_01', name: 'Read')
content_block_delta  (delta.type: 'input_json_delta', delta.partial_json: '{"file')
content_block_delta  (delta.type: 'input_json_delta', delta.partial_json: '_path":"/tmp/a"}')
content_block_stop   → 此时拼接 partial_json，JSON.parse() 得到完整 input
```

打开 `references/claude-code/src/services/api/claude.ts`。

这是 Claude Code 最核心的文件之一。它处理了流式事件、工具调用、多种 provider（Bedrock / Vertex / Foundry）、重试、token 统计等复杂逻辑。

**对我们有用的部分：** 搜索 `content_block_start`、`content_block_delta`、`content_block_stop`，看它如何把流式事件转换成结构化的消息。

我们的目标是实现同样的核心逻辑，但去掉所有多余的复杂度。

## 动手实现

创建 `src/anthropic.ts`，实现一个 `callAnthropic()` 函数。

**函数签名（你来决定具体实现）：**

```ts
export async function* callAnthropic(
  messages: Message[],
  tools: Tool[],
  systemPrompt: string,
  model: string,
): AsyncGenerator</* 你来决定 yield 什么 */>
```

**这个函数需要做到：**

1. 把我们的 `Message[]` 格式转换成 Anthropic SDK 需要的格式
   - `ToolResultBlock` 需要转成 `{ type: 'tool_result', tool_use_id: ..., content: ... }`
   - 注意 Anthropic API 的 `tool_result` 里 `is_error` 字段名与我们内部略有不同

2. 把我们的 `Tool[]` 转成 Anthropic SDK 的 tools 格式
   - `name`、`description` 直接用
   - `input_schema` 需要从 Zod schema 生成 JSON Schema
   - Zod v4 提供 `z.toJSONSchema(schema)` 方法

3. 调用 `client.messages.stream({ model, max_tokens, system, messages, tools })`

4. 处理流式事件，yield 出统一的事件对象
   - 文本增量：yield `{ type: 'text_delta', text: '...' }`
   - 工具调用：在 `content_block_stop` 时 yield `{ type: 'tool_use', id, name, input }`

**提示：** 工具调用的 `input` 是通过拼接多个 `input_json_delta` 事件的 `partial_json` 字符串得到的，最后 `JSON.parse()` 拿到完整对象。需要在函数内部维护临时变量来累积这个字符串。

**API Key 从哪来：** 暂时从环境变量 `ANTHROPIC_API_KEY` 读取，初始化 `new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })`。

## 验证

在 `bin/index.ts` 里临时写一段测试代码，直接调用 `callAnthropic()`：

```bash
ANTHROPIC_API_KEY=sk-ant-... bun run bin/index.ts
```

能看到 Claude 的流式回复逐字输出，说明接入成功。
