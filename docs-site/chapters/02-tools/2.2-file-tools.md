# 2.2 文件工具

## 背景与目标

Coding Agent 最基础的能力：**读文件、写文件、看文件片段**。没有这三个工具，Agent 看不到代码，也无法修改任何东西。

**本节结束后，你应该有：**
- `src/tools/read-file.ts` — `Read` 工具
- `src/tools/write-file.ts` — `Write` 工具
- `src/tools/view-range.ts` — `ViewRange` 工具

---

## Claude Code 是怎么做的

Claude Code 的文件工具在 `references/claude-code/src/tools/` 目录下，每个工具一个目录。

重点参考：
- `src/tools/ReadTool/` — 文件读取，注意它的 `maxResultSizeChars` 限制
- `src/tools/FileEditTool/` — 文件编辑（我们的 WriteFile 比它简单得多）

Claude Code 的工具比我们复杂很多（有权限检查、有 UI 渲染、有搜索高亮等），但核心功能逻辑是相通的。

## 动手实现

### `src/tools/read-file.ts` — `Read` 工具

**功能：** 读取文件的完整内容。

**inputSchema：**
- `file_path: string` — 文件的绝对路径

**实现要点：**
- 用 Node.js `fs.readFileSync(path, 'utf-8')` 读取
- 读取失败（文件不存在等）→ 返回 `{ content: '错误信息', isError: true }`，不要 throw
- `isReadOnly: true`

**工具名：** `Read`（和 Claude Code 保持一致，LLM 已经学过这个工具名）

---

### `src/tools/write-file.ts` — `Write` 工具

**功能：** 将内容写入文件。如果文件不存在则创建，父目录不存在则自动创建。

**inputSchema：**
- `file_path: string` — 目标文件绝对路径
- `content: string` — 要写入的内容

**实现要点：**
- 用 `fs.mkdirSync(dirname(file_path), { recursive: true })` 确保父目录存在
- 用 `fs.writeFileSync(file_path, content, 'utf-8')` 写入
- 成功 → 返回 `{ content: 'File written: /path/to/file' }`
- `isReadOnly: false`

---

### `src/tools/view-range.ts` — `ViewRange` 工具

**功能：** 读取文件的指定行范围（1-indexed，闭区间）。

查看大文件时，Agent 不需要读整个文件，只需要看相关的几十行。这是对 `Read` 的重要补充。

**inputSchema：**
- `file_path: string`
- `start_line: number` — 起始行（从 1 开始）
- `end_line: number` — 结束行（包含）

**实现要点：**
- 读取文件内容，按 `\n` 分割成行数组
- 取 `lines.slice(start_line - 1, end_line)` 再 `join('\n')`
- `isReadOnly: true`

**在 `ALL_TOOLS` 里注册这三个工具。**

## 验证

写 `src/tools/file-tools.test.ts`，使用临时目录测试：

```ts
const TMP = '/tmp/steven-test-' + Date.now()
// beforeEach: mkdirSync(TMP, { recursive: true })
// afterEach:  rmSync(TMP, { recursive: true, force: true })
```

测试覆盖：
- `Read`：正常读取 / 不存在的文件返回 isError
- `Write`：创建新文件 / 自动创建父目录
- `ViewRange`：正确返回指定行范围（验证 1-indexed 是否正确）

然后用真实 CLI 测试：
```bash
ANTHROPIC_API_KEY=sk-ant-... bun run bin/index.ts -p "读取 src/core/query.ts 并告诉我它的大致结构"
```
