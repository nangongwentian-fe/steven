# 2.3 系统工具

## 背景与目标

文件工具让 Agent 能读写文件，系统工具让 Agent 能**探索项目结构、搜索代码、执行命令**。

- `src/tools/run-command.ts` — `Bash` 工具
- `src/tools/list-dir.ts` — `LS` 工具

## 动手实现

### `src/tools/run-command.ts` — `Bash` 工具

**功能：** 执行 shell 命令，返回 stdout；失败时返回 stderr。

**inputSchema：**
- `command: string` — 要执行的 shell 命令

**实现要点：**

用 Bun 的 `Bun.spawnSync()`，而不是 Node.js 的 `child_process.execSync()`：

```ts
const proc = Bun.spawnSync(['sh', '-c', command], {
  stdout: 'pipe',
  stderr: 'pipe',
})
```

- `proc.exitCode !== 0` → 返回 `{ content: stderr 内容, isError: true }`
- 成功 → 返回 `{ content: stdout 内容 }`
- `isReadOnly: false`（执行命令可能修改系统状态）

**工具名：** `Bash`

---

### `src/tools/list-dir.ts` — `LS` 工具

**功能：** 列出目录内容，区分文件和子目录（子目录名后面加 `/`）。

**inputSchema：**
- `path: string` — 要列出的目录绝对路径

**实现要点：**
- 用 `fs.readdirSync(path)` 获取条目列表
- 用 `fs.statSync(join(path, name)).isDirectory()` 判断是否目录
- 目录名加 `/` 后缀，文件名不加
- 按名称排序，所有条目 `join('\n')` 返回
- `isReadOnly: true`

**工具名：** `LS`

---

### `src/tools/grep.ts` — `Grep` 工具

**功能：** 在指定目录（或文件）中搜索匹配正则的行，返回 `文件名:行号:匹配内容` 格式。

**inputSchema：**
- `pattern: string` — 正则表达式
- `path: string` — 搜索目标（目录或文件）

**实现要点：**

直接调用系统的 `grep` 命令（简单且高效）：

```ts
Bun.spawnSync(['grep', '-rn', pattern, path], { stdout: 'pipe', stderr: 'pipe' })
```

- 有输出 → 返回匹配结果
- 无输出（没有匹配）→ 返回 `{ content: 'No matches found.' }`（注意：这不是错误）
- `isReadOnly: true`

**工具名：** `Grep`

### 更新 `ALL_TOOLS`

在 `src/tools/registry.ts` 的 `ALL_TOOLS` 里加入全部 6 个工具：

```ts
export const ALL_TOOLS: Tool[] = [
  readFileTool,    // isReadOnly: true
  viewRangeTool,   // isReadOnly: true
  listDirTool,     // isReadOnly: true
  grepTool,        // isReadOnly: true
  writeFileTool,   // isReadOnly: false
  runCommandTool,  // isReadOnly: false
]
```

顺序建议：只读工具在前，写工具在后——方便 LLM 的 tool list 阅读。

## （可选）优化：工具并发执行

现在 `runTools()` 是串行的，可以优化：

把 `toolUseBlocks` 分成两组：
- `readOnly`（`isReadOnly === true`）→ `Promise.all()` 并发执行
- `writes`（`isReadOnly === false`）→ 串行执行

**注意：** 先执行完所有只读工具，再执行写工具。不要混合顺序。

参考：`references/claude-code/src/services/tools/toolOrchestration.ts` 的 `partitionToolCalls()` 函数。
