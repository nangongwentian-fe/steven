# 2.1 defineTool 工厂

## 背景与目标

在实现具体工具之前，先建立"工具是怎么定义的"这个约定——一个工厂函数 `defineTool()`，所有工具都通过它创建，保证接口一致、默认值安全。

**本节结束后，你应该有：**
- `src/tools/registry.ts`：`defineTool()` 工厂函数 + `ALL_TOOLS` 注册表

---

## Claude Code 的 buildTool() 模式

打开 `references/claude-code/src/Tool.ts`，拉到最底部，找到 `buildTool()` 函数（大约第 757 行）。

这个函数做的事很简单：**把一组默认值和用户的定义合并**。

```ts
const TOOL_DEFAULTS = {
  isEnabled: () => true,
  isConcurrencySafe: (_input?) => false,  // 默认：不安全（保守）
  isReadOnly: (_input?) => false,          // 默认：写操作（保守）
  checkPermissions: (...) => Promise.resolve({ behavior: 'allow' }),
  // ...
}

export function buildTool<D extends AnyToolDef>(def: D): BuiltTool<D> {
  return { ...TOOL_DEFAULTS, userFacingName: () => def.name, ...def }
}
```

**设计精髓：** 默认值都是**保守**的（assume write、assume not safe）。如果工具没有声明 `isReadOnly: true`，就按写操作处理。宁可多限制，不要误放行。

## 动手实现

创建 `src/tools/registry.ts`。

### `defineTool()` 工厂函数

参考 `buildTool()` 的思路，实现一个更简单的版本：

- 接受 `ToolDef<TInput>`（`isReadOnly` 可选）
- 返回 `Tool<TInput>`（`isReadOnly` 必须有值）
- 默认值：`isReadOnly: false`（保守原则）

```ts
export function defineTool<TInput>(def: ToolDef<TInput>): Tool<TInput> {
  return {
    isReadOnly: false,
    ...def,
  }
}
```

### `ALL_TOOLS` 注册表

在 registry.ts 底部导出所有工具的集合：

```ts
export const ALL_TOOLS: Tool[] = [

```

`bin/index.ts` 里创建 `Engine` 时传入 `ALL_TOOLS`，之后新增工具只需要在这里加一行。

## 工具并发策略预告

`isReadOnly` 这个字段现在看起来只是个标记，但它影响工具的执行策略：

- `isReadOnly: true`（Read、LS、Grep、ViewRange）→ 可以**并发执行**
- `isReadOnly: false`（Write、Bash）→ **串行执行**

第一章我们用串行实现了 `runTools()`。这节之后，你可以选择优化 `runTools()`，把并发策略加进去，参考 `references/claude-code/src/services/tools/toolOrchestration.ts`。
